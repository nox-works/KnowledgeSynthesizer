# Quality Criteria for Knowledge Synthesis

## Pre-Delivery Checklist

### Truth & Accuracy
- [ ] Every substantive claim is accurate to best current knowledge
- [ ] Uncertainty is explicitly labeled ("likely", "probably", "unknown")
- [ ] Assumptions are named and distinguished from facts
- [ ] Sources are cited for specific factual claims
- [ ] Models are distinguished from reality (frameworks ≠ truth)

### Clarity & Comprehension
- [ ] Core truth is extractable from first paragraph/section
- [ ] No sentence requires re-reading to understand
- [ ] Simplest language that captures full depth
- [ ] Technical terms are defined on first use
- [ ] Structure matches the content (not all topics fit all structures)

### Depth & Rigor
- [ ] Analysis pushes to first principles before building up
- [ ] Multiple perspectives are genuinely considered (not listed then ignored)
- [ ] Counterarguments are addressed, not suppressed
- [ ] Second and third-order implications are explored
- [ ] Knowledge gap between surface understanding and genuine depth is bridged

### Zero Bullshit
- [ ] No filler phrases ("great question", "let me share", "it's important to note")
- [ ] No softballing of uncomfortable truths
- [ ] No weasel words ("some say", "research suggests" without specifics)
- [ ] No false certainty — "I don't know" when you don't know
- [ ] Every sentence serves a purpose or is removed

### Variance (if applicable)
- [ ] Each variant is genuinely different, not restated content
- [ ] Each variant is internally coherent and self-contained
- [ ] Core truth is consistent across variants
- [ ] Variants are clearly labeled (reader knows what they're getting)
- [ ] Depth variance truly changes complexity, not just length

### Usefulness
- [ ] Reader can DO something with this knowledge
- [ ] Implications and applications are clear
- [ ] Next steps or recommendations are provided if applicable
- [ ] Knowledge is organized for retrieval, not just reading

## Common Synthesis Errors to Avoid

| Error | Symptom | Fix |
|-------|---------|-----|
| **Shallow analysis** | Stops at first-order answers | Push to first principles |
| **False certainty** | Overconfident claims | Label confidence levels |
| **Missing perspective** | Only one viewpoint | Scan for what lens is absent |
| **Structure mismatch** | Guide format for philosophical topic | Match structure to content |
| **Filler content** | Paragraphs that add nothing | Cut ruthlessly |
| **False variance** | Variants that are the same | Find genuinely different angles |
| **Oversimplification** | Complex topic reduced to soundbite | Feynman test: simple but not simpler |
| **Complexity without value** | Overcomplicated explanations | Cut everything that doesn't serve understanding |
